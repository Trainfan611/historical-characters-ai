# 🔧 Исправление невалидного токена бота

## ❌ Проблема: `{"ok":false,"error_code":404,"description":"Not Found"}`

Это означает, что токен бота неправильный или бот не существует.

## 🔍 Пошаговая диагностика

### Шаг 1: Проверьте токен в Railway

1. Откройте **Railway** → Ваш проект
2. Перейдите в **Variables**
3. Найдите `TELEGRAM_BOT_TOKEN_BASE` или `TELEGRAM_BOT_TOKEN`
4. **Проверьте формат:**
   - ✅ Должен быть: `1234567890:ABCdefGHIjklMNOpqrsTUVwxyz`
   - ❌ НЕ должен быть: пустым, с пробелами, обрезанным

### Шаг 2: Проверьте бота в BotFather

1. Откройте [@BotFather](https://t.me/BotFather) в Telegram
2. Отправьте команду `/mybots`
3. Посмотрите список ваших ботов

**Если бот НЕ найден:**
- Бот был удален
- Нужно создать нового бота

**Если бот найден:**
- Получите токен заново (см. ниже)

## ✅ Решение 1: Получить токен заново

### Если бот существует:

1. В [@BotFather](https://t.me/BotFather) отправьте `/mybots`
2. Выберите вашего бота
3. Выберите **API Token**
4. Скопируйте токен (формат: `1234567890:ABCdefGHIjklMNOpqrsTUVwxyz`)
5. Обновите в Railway:
   - Перейдите в **Variables**
   - Найдите `TELEGRAM_BOT_TOKEN_BASE` или `TELEGRAM_BOT_TOKEN`
   - Вставьте новый токен
   - Сохраните

### Если бот НЕ существует:

1. В [@BotFather](https://t.me/BotFather) отправьте `/newbot`
2. Следуйте инструкциям:
   - Введите имя бота (например: "Historical Characters Bot")
   - Введите username бота (например: "history_characters_bot")
3. Получите токен (формат: `1234567890:ABCdefGHIjklMNOpqrsTUVwxyz`)
4. Обновите в Railway:
   - Перейдите в **Variables**
   - Создайте или обновите `TELEGRAM_BOT_TOKEN_BASE`
   - Вставьте новый токен
   - Сохраните

## ✅ Решение 2: Проверьте формат токена

Токен должен быть в формате:
```
ЧИСЛА:БУКВЫ_И_СИМВОЛЫ
```

**Пример правильного токена:**
```
1234567890:ABCdefGHIjklMNOpqrsTUVwxyz-1234567890
```

**Проверьте:**
- ✅ Начинается с цифр
- ✅ Есть двоеточие `:`
- ✅ После двоеточия буквы, цифры, дефисы, подчеркивания
- ✅ Нет пробелов
- ✅ Длина обычно 40-50 символов

## 🔧 После получения правильного токена

### 1. Проверьте токен

Откройте в браузере (замените `НОВЫЙ_ТОКЕН`):
```
https://api.telegram.org/botНОВЫЙ_ТОКЕН/getMe
```

**Должно вернуться:**
```json
{
  "ok": true,
  "result": {
    "id": 123456789,
    "is_bot": true,
    "first_name": "Your Bot",
    "username": "your_bot"
  }
}
```

### 2. Обновите webhook

Откройте в браузере (замените `НОВЫЙ_ТОКЕН`):
```
https://api.telegram.org/botНОВЫЙ_ТОКЕН/setWebhook?url=https://history-character.up.railway.app/api/telegram/webhook
```

**Должно вернуться:**
```json
{
  "ok": true,
  "result": true,
  "description": "Webhook was set"
}
```

### 3. Проверьте webhook

Откройте в браузере (замените `НОВЫЙ_ТОКЕН`):
```
https://api.telegram.org/botНОВЫЙ_ТОКЕН/getWebhookInfo
```

**Должно быть:**
- `result.url` = `https://history-character.up.railway.app/api/telegram/webhook`
- `result.pending_update_count` = 0

## 💡 Важные замечания

1. **Токен чувствителен к регистру** - скопируйте точно
2. **Не добавляйте пробелы** - токен должен быть одной строкой
3. **Не обрезайте токен** - скопируйте полностью
4. **Токен должен начинаться с цифр** - формат: `ЧИСЛА:СИМВОЛЫ`

## 🆘 Если ничего не помогает

1. Создайте нового бота через `/newbot` в BotFather
2. Получите новый токен
3. Обновите в Railway
4. Настройте webhook заново
5. Обновите `NEXT_PUBLIC_TELEGRAM_BOT_USERNAME` в Railway (имя бота без @)
