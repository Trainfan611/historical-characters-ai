# 🔧 Решение проблемы доступа к Nano Banana API

Если вы видите ошибку:
```
Nano Banana API access denied: You do not have access permissions
```

Это означает, что ваш API ключ Nano Banana не имеет доступа к функции генерации изображений.

## ✅ Решение

### Шаг 1: Проверьте API ключ

1. Откройте https://nanobananaapi.ai/api-key
2. Убедитесь, что ваш API ключ активен
3. Проверьте, что ключ имеет доступ к функции **Image Generation**

### Шаг 2: Обновите ключ в Railway

1. Откройте проект в Railway
2. Перейдите в **Variables**
3. Найдите `NANO_BANANA_API_KEY`
4. Обновите значение на новый ключ (если получили новый)
5. Сохраните изменения

### Шаг 3: Проверьте работу

После обновления ключа система автоматически попробует использовать Nano Banana при следующей генерации.

---

## 🔄 Автоматический Fallback

**Хорошая новость:** Даже если Nano Banana недоступен, система автоматически переключится на другие провайдеры:

1. **Nano Banana** (первый приоритет) ❌ → 
2. **OpenAI gpt-image-1** (fallback) ✅ → 
3. **Replicate Flux** (финальный fallback) ✅

Генерация изображений будет работать, просто будет использоваться другой провайдер.

---

## 📝 Что делать, если проблема не решена

1. **Проверьте баланс на аккаунте Nano Banana**
   - Убедитесь, что на счету есть средства
   - Проверьте лимиты использования

2. **Создайте новый API ключ**
   - Войдите в https://nanobananaapi.ai/api-key
   - Создайте новый ключ
   - Обновите `NANO_BANANA_API_KEY` в Railway

3. **Проверьте логи Railway**
   - Откройте логи последнего деплоя
   - Найдите сообщения с префиксом `[Nano Banana]`
   - Проверьте, что ключ правильно установлен

---

## 💡 Рекомендации

- Если Nano Banana постоянно недоступен, можно временно отключить его, удалив `NANO_BANANA_API_KEY` из Railway
- Система автоматически будет использовать OpenAI gpt-image-1 как основной провайдер
- Когда Nano Banana снова заработает, просто добавьте ключ обратно
