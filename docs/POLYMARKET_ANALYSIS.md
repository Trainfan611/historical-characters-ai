# Анализ иррациональных рынков на Polymarket

## Описание

Скрипт для поиска рынков на Polymarket, которые имеют иррациональные цены (завышенные или заниженные) в течение дня.

## Использование

```bash
npm run polymarket:irrational
```

или

```bash
tsx scripts/find-irrational-polymarket.ts
```

## Критерии иррациональности

Скрипт анализирует рынки по следующим критериям:

1. **Экстремальные цены** (score +35)
   - Цена < 5% или > 95% при объеме > $500
   - Критические цены < 2% или > 98% (дополнительно +20)

2. **Низкая ликвидность** (score +25)
   - Ликвидность < $100 при экстремальной цене

3. **Низкий объем** (score +20)
   - Объем < $500 при экстремальной цене

4. **Высокий объем при подозрительной цене** (score +15)
   - Объем > $10,000 при цене < 10% или > 90% (возможная манипуляция)

5. **Неопределенность при высоком объеме** (score +15)
   - Цены исходов очень близки (разница < 10%) при объеме > $5,000

6. **Дисбаланс объемов** (score +10)
   - Сильный дисбаланс объемов между исходами

## Результаты

Скрипт выводит:
- Список иррациональных рынков с детальной информацией
- Оценку иррациональности (0-100)
- Причины иррациональности
- Статистику по всем рынкам

Результаты также сохраняются в JSON файл в папке `data/` с временной меткой.

## Формат вывода

Для каждого иррационального рынка выводится:
- Вопрос рынка
- Ссылка на Polymarket
- Текущая цена
- Объем торгов
- Ликвидность
- Оценка иррациональности
- Причины (завышена/занижена/аномалия)
- Категория
- Дата окончания

## Примечания

- Скрипт пытается подключиться к Polymarket API через несколько эндпоинтов
- Если API недоступен, скрипт использует тестовые данные для демонстрации
- Для работы с реальными данными Polymarket:
  - Проверьте официальную документацию: https://docs.polymarket.com/
  - Возможно, требуется API ключ или специальная аутентификация
  - Используйте официальный Polymarket SDK или GraphQL клиент
- Порог иррациональности можно изменить в коде (по умолчанию > 15)

## Получение реальных данных

Для работы с реальным Polymarket API вам нужно:

1. **Изучить официальную документацию**: https://docs.polymarket.com/
2. **Использовать правильный GraphQL endpoint** (может требовать аутентификацию)
3. **Или использовать веб-скрапинг** (не рекомендуется, может нарушать ToS)
4. **Или использовать официальный SDK** Polymarket, если доступен

Текущая версия скрипта использует тестовые данные для демонстрации логики анализа.
